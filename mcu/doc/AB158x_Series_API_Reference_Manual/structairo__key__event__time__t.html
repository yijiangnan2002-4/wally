Detailed Description

The event time configuration structure of the airokey framework, the unit is millisecond.

Field Documentation

uint32_t t_long_press_1

The first level of long press time.

uint32_t t_long_press_2

The second level of long press time.

uint32_t t_long_press_3

The third level of long press time.

uint32_t t_press

The press time of the click.


In the case of a multiple click, if the long press event is supported, the first press time is t_long_press_1. If the short long press event is supported, the second press time is t_slong. Otherwise, t_press will be used.

uint32_t t_release

The release time is the period of time to wait for the next click, only for multiple click cases.

uint32_t t_repeat

The repeat event time.


If repeat event is supported, it is reported after long press event or short long press event.

uint32_t t_silence

The silence time, only for click cases.


For example, if a key only supports double-click, when the double-click event is sent, it will enter the silence time. During silence time, the driver will not process any unnecessary clicks. The silence time restarts when a new release occurs.
The key can only enter the next event state when the silence time expires.

uint32_t t_slong

The short long press time.
