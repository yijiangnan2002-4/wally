Overview

This section introduces the Infrared Receiver(IRRX) APIs including terms and acronyms, supported features, software architecture, details on how to use this driver, IRRX function groups, enums, structures and functions.

Terms and acronyms

Terms Details
IRRX Infrared Receiver that detects infrared radiation. For more information, please refer to introduction to IrDA in Wikipedia .

Supported features

  • Support RC5 NEC RC6 RCMM and PWD modes. The legacy design supports two hardware decoding methods(RC5,pwd),and the new design supports four decoding methods(NEC,RC5,RC6,RCMM):
    • NEC mode: The NEC protocol uses pulse distance encoding of the bits.
      In this mode, the NEC code can be received and decoded directly.The first bit IRRX sample would be the bit that the arrow point. The bit order would be auto adjust by hardware.
    • RC5 mode: The Philips RC5 IR transmission protocol uses Manchester encoding of the message bits.
      In this mode, the RC5 code can be received and decoded directly.The first bit IRRX sample would be the bit that the arrow point. The bit order would be auto adjust by hardware.
    • RC6 mode: The Philips RC6 IR transmission protocol uses Manchester encoding of the message bits. In this mode, the RC6 code can be received and decoded directly.The RC6 potocol is similar with RC5.
    • RCMM mode: The Philips RCMM IR transmission protocol uses Manchester encoding of the message bits.
      In this mode, the RCMM code can be received and decoded directly.The first bit IRRX sample would be the bit that the arrow point. The bit order would be auto adjust by hardware.
    • PWD mode: PWD is pulse-width-detection in short. In this mode, the hardware detects the wavelengths, any type of code transmitted by infrared radiation can be received.The received content could be determined by using the wavelength.

How to use this driver

  • Use IRRX in the NEC mode.
  • Step1: Call hal_gpio_init() to init the pins, if EPT tool hasn't been used to configure the related pinmux.
  • Step2: Set GPIO34 to RX function to enable the IRRX mode.
  • Step3: Call hal_irrx_init() to initialize the IRRX.
  • Step4: Call hal_irrx_receive_nec_start() to start the hardware.
  • Step5: Call hal_irrx_receive_nec() to receive the NEC code.
  • Step6: Call hal_irrx_deinit() to deinitialize the IRRX.
  • sample code:
    //Init GPIO, set GPIO pinmux(if EPT tool hasn't been used to configure the related pinmux).
    hal_gpio_init(HAL_GPIO_34);
    //Call hal_pinmux_set_function() to set GPIO pinmux, for more information, please reference hal_pinmux_define.h
    //function_index = HAL_GPIO_34_IR_RX.Need not to configure pinmux if EPT tool is used.
    hal_pinmux_set_function(HAL_GPIO_34, function_index); //Sets the GPIO34 to IRRX mode.
    hal_irrx_init();
    hal_irrx_receive_nec_start(user_callback,NULL); //A callback function for the hardware will be triggered when the received code is decoded.
    // Callback function. This function should be registered with #hal_irrx_receive_nec_start().
    void user_callback (void *user_data)
    {
    hal_irrx_receive_nec(data,bit_length);
    hal_irrx_deinit();
    // user's handler
    }
