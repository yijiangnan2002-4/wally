Overview

This section introduces the ui_shell interface APIs including terms and acronyms, supported features, software architecture, details on how to use this interface, ui_shell function groups, enums, structures and functions.

Terms and acronyms

Terms Details
Activity A structure that must be implemented by the AP layer to process events.
Pre-proc Activity A type of activity that is unique in a project. It pre-processes all events.
Idle Activity A type of activity that is normally created when the system starts.
Transient Activity A type of activity that is created when running.

Send an allowance request to the UI shell.

The UI shell sends it to all active activities. The activities can return true to immediately allow it, or use ui_shell_grant_allowance() to allow it later.

Parameters

void(*)(void)  special_free_extra_func,
uint32_t  delay_ms 
)

Send an event to the UI shell.

The UI shell dispatches it to activities after a delay.

Parameters
[in]from_isrmust be set to true if it calls in ISR.
[in]priorityis the priority of the event.
[in]event_groupis the group value of the event.
[in]event_idis the id of the event.
[in]datais the extra data that is sent to activities with the event.
[in]data_lenis the length of data. When data_len > 0, the UI automatically frees the data.
[in]special_free_extra_funcis useful when data_len > 0. The UI shell calls it before it frees data.
[in]delay_msis the delay time to dispatch the event to activities.
Returns
UI_SHELL_STATUS_INVALID_STATE if the UI shell has not started. UI_SHELL_STATUS_INVALID_PARAMETER if the priority is out of range.

ui_shell_status_t ui_shell_start_activity ( ui_shell_activity_t *  self,
ui_shell_proc_event_func_t  proc_event,
ui_shell_activity_priority_t  priority,
void *  extra_data,
size_t  data_len 
)

start an activity

Parameters
[in]selfis the caller of the functions. The value could be NULL when idle activities are started by the system during the system initialization stage.
[in]proc_eventis the proc event callback of the target activity that must be started.
[in]priorityis the priority of the target activity to be started.
[in]extra_datais the data that is received with the ON_CREATE event by target activity.
[in]data_lenis the data length of the extra data. If data_len > 0, the UI shell automatically frees up extra data.
Returns
UI_SHELL_STATUS_INVALID_STATE if the UI shell has not started and the priority is greater than ACTIVITY_PRIORITY_IDLE_TOP. UI_SHELL_STATUS_INVALID_PARAMETER if proc_event is NULL or the priority is out of range UI_SHELL_STATUS_OUT_OF_RESOURCE if it cannot create a new activity.
