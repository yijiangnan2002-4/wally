Introduction

Overview

The Race Command (RCMD) packet is used to send commands to Airoha IoT SDK for BT-Audio device from the host (external MCU or PC tool) or receive events (notifications or responses) from Airoha IoT SDK for BT-Audio device. Any Airoha IoT SDK for BT-Audio device is able to accept RCMD with up to 1000 bytes of data excluding the RCMD header and length field. Each RCMD command is assigned two types of transported used to uniquely identify different formats of commands. These two fields called the Transported by H4 and Transported by H5.

RACE packet format

Field name Bytes Example Description
RACE indicator 1 byte 0x05 Fixed value 0x05
RACE type 1 byte 0x5A Four RACE types: 0x5A, 0x5B, 0x5C, 0x5D
Length 2 bytes 0x03 0x00 Length in little endian
RACE ID 2 bytes 0x00 0x0A RACE ID in little endian
Payload N bytes 0x01 Each RACE ID defines its payload
