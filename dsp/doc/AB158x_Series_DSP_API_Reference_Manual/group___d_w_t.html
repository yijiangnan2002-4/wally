Overview

This section introduces the DSP Debug Watchpoint and Trace(DWT) APIs including terms and acronyms, supported features, software architecture, details on how to use this driver, enums, structures and functions.

Terms and acronyms

Terms Details
DWT DWT is the Debug Watchpoint and Trace Unit of DSP. For more details.

Supported features

  • Support allocate a DWT channel for watchpoints.
    Allocate a DWT channel by hal_dwt_request_watchpoint()
    For address matching, the comparator can use a mask, so it can match a specific address and also support match a range of addresses.
    It will generate a DebugMonitor exception on a successful match.

  • Support enable and disable the DWT channel.
    Enable or disable the DWT channel by hal_dwt_control_watchpoint()

How to use this driver

  • step1: Masking DWT channel number that you want to use,You need to mask the line of funtion hal_dwt_request_watchpoint(channel1 or channel2) in port.c file
  • step2: call hal_dwt_init() to init the DWT unit.
  • step3: call hal_dwt_request_watchpoint() to allocate a DWT channel masked in step1 and start the watchpoints.
  • step4: call hal_dwt_control_watchpoint() to enable or disable the DWT channel if needed.
  • sample code:
    int ret;
    uint32_t mask;
    static uint32_t check_addr = 0x0;
    hal_dwt_init();
    //allocate DWT channel 0 to monitor the address range [0, (2^3-1)].
    //the start_addr must be aligned with 2^3.
    //only write access(WDE_DATA_WO) to the specific address range [0, (2^3-1)] will generate a DebugMonitor exception.
    ret = hal_dwt_request_watchpoint(HAL_DWT_0, (uint32_t)&check_addr, 0x3, WDE_DATA_WO);
    if(0 != ret) {
    //error handle
    }
    //#hal_dwt_control_watchpoint() can be used to enable or disable the monitor.
    hal_nvic_save_and_set_interrupt_mask(&mask);
    hal_dwt_control_watchpoint(HAL_DWT_0, HAL_DWT_DISABLE); //disable the monitor.
    check_addr = 0x1; //not update the value,the check_addr was the pass second parameter in the last hal_dwt_request_watchpoint called.
    hal_dwt_control_watchpoint(HAL_DWT_0, HAL_DWT_ENABLE); //re-start the monitor.
    hal_nvic_restore_interrupt_mask(mask);

Allocate a DWT channel and monitor specific a range of addresses.

Parameters
[in]indexis the channel that will be allocated. This parameter can only be the value of type hal_dwt_channel_t.
[in]addr_baseis the start address to be monitored, the addr_base must be aligned with the 2^addr_mask
[in]addr_maskis the ignore mask applied to address range matching
[in]funcis which kind of compared accesses will generate watchpoint debug event. This parameter can only be the value of type hal_dwt_function_t
Returns
Allocate channel status, -1: fail; 0: success
Note
DWT channel 0 and DWT channel 1 already used ,if you want to use,you need mask the channel follow demo code step1.
