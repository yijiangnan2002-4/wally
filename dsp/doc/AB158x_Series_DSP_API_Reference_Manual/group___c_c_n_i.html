Overview

This section introduces the Cross Core Notification Interface(CCNI) HAL APIs including terms and acronyms, details on how to use this driver, supported features, enums, structures and functions.

This section introduces how to define a CCNI event.

Every CCNI event is assigned to a specific user, the event handler routine is predefined. The ccni driver will be initialized at system initiation stage.

Terms and acronyms

Terms Details
CCNI Cross Core notification interface

How to use this driver

  • How to configure a CCNI event
    • It looks at bits 0 to 31 as events 0 to 31, keeps events 0 to 7 used with a message and events 8 to 31 used without a message.
    • We strongly advise that you do not change the default configuration.
      (A) There are some events without a message which had been used as system default.
      (B) There are events with a message which uses reserved memeory.
  • How to define an event
    The CCNI module had 32 events between any two cores. The driver defines a special name for every event in hal_ccni_config.h for easy use.
    • Step 1: Decide to use an event with or without a message.
    • Step 2: Find an event that is not used.
      (A) Check the event usage status in \dsp\driver\chip\*\dsp0\inc\hal_ccni_config.h. The event in default status can be used.
      (B) Or check the ccni_*_to_*_function_table[] in hal_ccni_config.c; The event is not used if the isr is default_isr.
      For example, if you want to apply an event for MCU to DSP0, please check ccni_cm4_to_dsp0_function_table[] to find an unused event.
      (C) If you want an event with a message, please select an event within event 0 to 7;
      (D) If you want an event without a message, please select an event within event 8 to 31.
    • Step 3: Replace the event name in \dsp\driver\chip\*\dsp0\inc\hal_ccni_config.h for DSP0.
      We strongly advise that you use a name which implies the event function.
      //define EVENT 8 for exception from DSP0 to MCU
      #define IRQGEN_DSP0_TO_CM4_EXCEPTION (HAL_CCNI_EVENT8|CCNI_EVENT_SRC_DSP0|CCNI_EVENT_DST_CM4)
  • Step 4: Implement isr handler.
    //sample code for isr handler
    void user_event_handler_routine(hal_ccni_event_t event, void *msg)
    {
    uint32_t *pMsg = msg;
    hal_ccni_status_t status;
    //mask event first
    status = hal_ccni_mask_event(event);
    // the msg points to the message address if the event is with message. Otherwise, the msg is equal to NULL.
    if( pMsg != NULL) {
    uint32_t address;
    //Suppose DSP0 received an EVENT from MCU with sharing memory and the memory address is in pMsg[0]. \n
    //Receiver should do memory address remap by hal_memview_infrasys_to_dsp0(), otherwise it can not get right data.
    hal_memview_infrasys_to_dsp0(pMsg[0]);
    log_hal_info("pMsg[0] = %x, pMsg[1] = %x \r\n", pMsg[0],pMsg[1]);
    }
    //user deal with the event!
    //If the event is managed here, please clear and unmask the event when it is complete
    //or the event will be managed at a thread level. Please clear and unmask the event when it is complete.
    //Otherwise, it cannot receive the event anymore.
    status = hal_ccni_clear_event(event); // clear the event.
    status = hal_ccni_unmask_event(event); // unmask the event.
    }
  • Step 5: Register the isr handler to ccni_*_to_*_function_table[] in \dsp\driver\chip\*\dsp0\src\hal_ccni_config.c for DSP0.
  • Use the CCNI driver to send an event without a message to the other core.
    • Step 1. Predefine the CCNI event and event handler routine in the source code.
    • Step 2. Call hal_ccni_set_event() to trigger an event.
    • Step 3. Should check the return value, and do error handling if return value is not HAL_CCNI_STATUS_OK.
  • Sample code:
    //DSP0 send an event IRQGEN_DSP0_TO_CM4_EVENT9 to MCU.
    uint32_t status;
    hal_ccni_event_t event = IRQGEN_DSP0_TO_CM4_EVENT9;
    // the second parameter should be NULL for event without message.
    ret = hal_ccni_set_event(event, NULL);
    if (HAL_CCNI_STATUS_OK != ret) {
    // Error handler;
    }
  • Using the CCNI driver to send an event with a message to other core.
    //DSP0 send an event CCNI_DSP0_TO_CM4_EVENT1 to MCU with const value in message.
    uint32_t status;
    //DSP0 send event CCNI_DSP0_TO_CM4_EVENT1 to MCU
    hal_ccni_event_t event = CCNI_DSP0_TO_CM4_EVENT1;
    uint32_t msg_array[2];
    //the destination core will receive the next 8-byte message.
    msg_array[0] = 0x80;
    msg_array[1] = 0xa5;
    //the second parameter should not be NULL.
    ret = hal_ccni_set_event(event, msg_array);
    if (HAL_CCNI_STATUS_OK != ret) {
    //Error handler;
    }
    //DSP0 send an event CCNI_DSP0_TO_CM4_EVENT2 to MCU and notify send 128 Bytes data with sharing memory.
    uint32_t status;
    uint32_t sharing_memory_address;
    //DSP0 send event CCNI_DSP0_TO_CM4_EVENT2 to MCU
    hal_ccni_event_t event = CCNI_DSP0_TO_CM4_EVENT2;
    uint32_t msg_array[2];
    //the destination core will receive the next 8-byte message.
    sharing_memory_address = 0xabc; // the memory 0xabc should be defined with #ATTR_SHARE_RWDATA.
    //Please use hal_memview_dsp0_to_infrasys() to do address remap.
    msg_array[0] = hal_memview_dsp0_to_infrasys(sharing_memory_address);
    //Please mind the receiver should call hal_memview_infrasys_to_mcu(msg_array[0]) to remap the address, otherwise the receiver can not get the right data.
    msg_array[1] = 0x80;
    //the second parameter should not be NULL.
    ret = hal_ccni_set_event(event, msg_array);
    if (HAL_CCNI_STATUS_OK != ret) {
    //Error handler;
    }
    // It is an example when got HAL_CCNI_STATUS_BUSY at #hal_ccni_set_event;
    uint32_t status;
    uint32_t count;
    do{
    status = hal_ccni_set_event(IRQGEN_DSP0_TO_CM4_EVENT9, NULL);
    if(status == HAL_CCNI_STATUS_BUSY) {
    hal_gpt_delay_ms(1);
    count++;
    }
    if (count > 10) {
    break; //wait a while and exit if it is overtime.
    }
    }while(status == HAL_CCNI_STATUS_BUSY);

hal_ccni_status_t hal_ccni_clear_event ( hal_ccni_event_t  event)

This function clear the event, it should be done by the event received core.

Parameters
[in]eventis the event name, please use the name define in hal_ccni_config.h.
Returns
HAL_CCNI_STATUS_OK means this function return successfully.
HAL_CCNI_STATUS_INVALID_PARAMETER means an invalid parameter is given by user.
HAL_CCNI_STATUS_BUSY means event is using by other module.
HAL_CCNI_STATUS_NO_INIT means CCNI module is not initiated.

hal_ccni_status_t hal_ccni_query_event_status ( hal_ccni_event_t  event,
uint32_t *  data 
)

This function use to query the event status.

Parameters
[in]eventis the event name, please use the name define in hal_ccni_config.h
[out]datakeep the query result, the data is valid only the function returns HAL_CCNI_STATUS_OK: HAL_CCNI_EVENT_STATUS_BUSY means event is busy.
HAL_CCNI_EVENT_STATUS_IDLE means event is idle.
Returns
HAL_CCNI_STATUS_OK means this function return successfully.
HAL_CCNI_STATUS_INVALID_PARAMETER means an invalid parameter is given by user.
HAL_CCNI_STATUS_BUSY means event is using by other module.
HAL_CCNI_STATUS_NO_INIT means CCNI module is not initiated.
Note
The function is only used to query the event status. Do not do hal_ccni_set_event directly when it returns HAL_CCNI_STATUS_BUSY as it has race condition risk.
hal_ccni_status_t hal_ccni_set_event ( hal_ccni_event_t  event,
hal_ccni_message_t *  message 
)

This function trigger an interrupt of the destination core.

Parameters
[in]eventis the event name, please use the name define in hal_ccni_config.h.
[in]messageis point which point to a eight bytes buffer when event is event 0 to 7. message should be NULL when event is event 8 to 31.
Returns
HAL_CCNI_STATUS_OK means this function return successfully.
HAL_CCNI_STATUS_INVALID_PARAMETER means an invalid parameter is given by user.
HAL_CCNI_STATUS_BUSY means event is using by other module.
HAL_CCNI_STATUS_NO_INIT means CCNI module is not initiated.
Note
Please mind if the function returns HAL_CCNI_STATUS_BUSY. It has race condition risk if it does hal_ccni_set_event after hal_ccni_query_event_status.
